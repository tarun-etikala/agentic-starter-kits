| Accelerator/Package Support  | Does this feature require support from the AIPCC team? Components that are pure golang should meet this requirement by default, since the binary is built as part of building the image. However, images with any other components need to be reviewed to ensure that: All RPMs are built by the RHEL or AIPCC teams and no EPEL, CentOS, or upstream packages from untrusted sources are used. All wheels are built by the AIPCC team, the image uses the AIPCC base images, and no content is installed from PyPI.org, <https://download.pytorch.org>, partner sites, or other external package indexes.
